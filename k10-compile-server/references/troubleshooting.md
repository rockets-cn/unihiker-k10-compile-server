# Troubleshooting

Common issues and their solutions.

## Server Won't Start

### Port 8900 already in use
```bash
sudo lsof -i :8900  # Find the process
sudo kill <PID>     # Kill it
# Or use a different port:
K10_COMPILE_PORT=8901 python3 -m uvicorn main:app --host 0.0.0.0 --port 8901
```

### "pio: command not found"
```bash
# PlatformIO not installed or not in PATH
python3 -m pip install platformio
# Find where pio is:
python3 -m platformio run --version
```

### "K10 toolchain not ready"
```bash
# Install the DFRobot UniHiker K10 Arduino platform
pio platform install https://github.com/DFRobot/UniHiker_K10_Arduino.git

# Or if pio is in a venv:
python3 -m platformio platform install https://github.com/DFRobot/UniHiker_K10_Arduino.git
```

## Compilation Fails

### "未找到 platformio.ini"
The upload doesn't contain a `platformio.ini`. Make sure your project includes one.

### "未找到源代码文件"
The server couldn't find `.cpp`, `.c`, or `.ino` files. Ensure your project has
a `src/` directory with source files, or that source files are in the project root.

### Compile timeout (>5 minutes)
Some first-time builds download toolchains which can take a while. On a fresh
PlatformIO install, the first compile may exceed the 300s timeout.

**Fix:** Increase `K10_COMPILE_TIMEOUT`:

```bash
export K10_COMPILE_TIMEOUT=600
# Docker: add to environment in docker-compose.yml
```

Or pre-install the toolchain:

```bash
# Pre-cache the ESP32-S3 toolchain
pio run --project-dir examples/Blink
```

### Out of disk space
Build artifacts accumulate in `/tmp/k10_compile_*` and `.pio/build/`.
The server automatically cleans up old builds, but if you're running many
compiles, you can manually clean:

```bash
# Clean PlatformIO build cache
rm -rf ~/.platformio/packages
# Clean temp build dirs
rm -rf /tmp/k10_compile_*
```

## Web Serial Issues

### "Please use Chrome/Edge over HTTPS"
Web Serial requires:
1. Chrome, Edge, or Opera (not Firefox, not Safari)
2. HTTPS connection (not HTTP, not `file://`)

**Workaround for LAN:**
- Use self-signed certificate (generated by `install.sh`)
- In Chrome, click "Advanced → Proceed to ... (unsafe)"
- Or type `thisisunsafe` on the cert error page (Chrome)

### No serial port found / "Failed to open port"
1. Make sure K10 is connected via USB
2. Try a different USB cable (some cables are charge-only)
3. On Linux, ensure you have permission: `sudo usermod -a -G dialout $USER`
   (log out and back in after)
4. On Windows, install CH340 driver

### Flash fails mid-way
1. Check the USB connection
2. Try a lower baudrate (esptool default 115200)
3. Hold BOOT button on K10, press RST, release BOOT, then try again
4. **K10's CH340 doesn't support hardware reset** — manual BOOT+RST is normal

### "Flash complete" but K10 doesn't boot
1. Press the RST button on K10
2. If still doesn't boot, the flash layout might be wrong:
   - `bootloader.bin` must be at `0x0`
   - `partitions.bin` must be at `0x8000`
   - `firmware.bin` must be at `0x10000`
3. Try erasing flash first: `esptool.py erase_flash` (server-side)

## Server-Side Flash Issues

### "未找到 esptool.py"
```bash
pip install esptool
```

### Permission denied on serial port
```bash
sudo usermod -a -G dialout $USER
# Log out and back in, or:
sudo chmod 666 /dev/ttyUSB0  # Temporary
```

## Docker Issues

### "docker: command not found"
Install Docker: https://docs.docker.com/engine/install/

### "docker compose" vs "docker-compose"
This project uses `docker compose` (v2, plugin syntax).
If you have the older `docker-compose` (v1), install the plugin:

```bash
sudo apt install docker-compose-plugin
# Or use: docker-compose up -d
```

### First Docker build takes a long time
The Docker image downloads PlatformIO and the UniHiker K10 toolchain.
This is a one-time cost — subsequent builds are fast thanks to Docker layer
caching and the `pio-data` volume.

### Permission errors on PlatformIO cache volume
```bash
# If running as non-root, ensure the volume is writable
docker compose down
docker volume rm k10-compile-server_pio-data
docker compose up -d
```

## General

### Build results expire too quickly
Default TTL is 300 seconds (5 minutes). Increase:

```bash
export K10_BUILD_TTL=1800  # 30 minutes
```

### Can't access server from another machine
1. Check firewall: `sudo ufw allow 8900/tcp`
2. Server binds to `0.0.0.0` by default (all interfaces)
3. If using Docker, port 8900 must be published (already in docker-compose.yml)
4. Check `ss -tlnp | grep 8900` to confirm it's listening
